==AWS作業メモ================================================
1. 金田さんが作ったAWSアカウントからログイン
https://051354606339.signin.aws.amazon.com/console
ユーザー名: yamaura
パス: OpenTerrace

2. トップからEC2をクリック
Create Instance の「Launch Instance」をクリック
OSにはAmazon Linuxを選択
Instance Typeは、一番上の「t2.micro」を選択
まだLaunchせず、Configure Instance Detailsにいって詳細設定、と思ったけど全部そのまま
さらに次へ進み、Add Storageの設定。そのままにした。8GiB
次のTag設定も特になにもせずそのまま
セキュリティの設定
 Create a new security groupを選択
 「sel-web-security」として設定
 SSHのみ、アクセス範囲をmyIP（@家）とした
 HTTP, HTTPSは任意の場所に設定
 既存のキーペアを選択して「denweb」を使用

3. Elastic IPの作成
myAWSのときとと同じように、作成後、associateで紐付け
52.68.240.189
として、設定完了

4. sshログイン
~/.ssh/にあるconfigを編集し、以下を追加した
# SEL webpage
Host selhp
  HostName 52.68.240.189
  Port 22
  User ec2-user
  IdentityFIle ~/.ssh/denweb.pem

5. 各種設定
sudo yum update　
Apacheインストール: sudo yum -y install httpd
Apache起動: sudo service httpd start
/var/www/html/の下にindex.htmlを作れば表示されるはず

6. ファイル転送
そのままだとhtmlのフォルダの権限が弱いので、「permission denied」となり転送出来ない

scp -i ~/.ssh/denweb.pem index.html ec2-user@52.68.240.189:/var/www/html
scp -i ~/.ssh/denweb.pem -r css ec2-user@52.68.240.189:/var/www/html
scp -i ~/.ssh/denweb.pem -r fonts ec2-user@52.68.240.189:/var/www/html
scp -i ~/.ssh/denweb.pem -r js ec2-user@52.68.240.189:/var/www/html
scp -i ~/.ssh/denweb.pem -r img ec2-user@52.68.240.189:/var/www/html

で、IPを入力すれば見れるようになった。

==================
git作業
https://github.com/ymrssk/selHP
1. GitHubにプライベートレポジトリを作成（有料アカウントになった）
2. 現状のローカルリポジトリを最新の状態にした（add, commit）
3. GitHubにレポジトリを作成した際に出てきた解説に従って、
git remote add origin https://github.com/ymrssk/selHP.git　
git push origin master　　
の２つのコマンドをローカルリポジトリで打ち、Hubに同期完了
4. サーバーにgitをインストール「sudo yum install git」
5. サーバー内、/etc/httpd/confのhttpd.confのファイルをvimで編集
DocumentRoot "/var/www/html"　
を
DocumentRoot "/var/www/html/selHP"　
に変更後
sudo apachectl restart　
でApacheを再起動させ、終了

==================
ファイル編集、更新までの流れ
1. ローカルで編集し、セーブ
2. git add, git commitで、ローカルのmasterブランチを更新
3. 「git push origin master」で、GitHubへアップ
4. サーバー側で、「git pull origin master」を打ち、更新完了
